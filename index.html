Order matters — you diagnose before you see the fix path, and you prompt before anything gets auto-solved for you.

STAGE 01

Diagnose

You're given a symptom only — no code, no stack trace. List 3–4 root causes you'd actually suspect, in order of likelihood. This is where domain experience shows.

Locate the fault

Prompt it

    // why this exists

    AI made code cheap.
    It made judgment expensive.

    Every junior dev now ships code an AI wrote. That code compiles, passes lint, and is often correct on the first try. The bottleneck moved — from typing the fix to knowing what's actually broken and telling the AI precisely enough to trust the output.

    1Code is built for that bottleneck. We never ask you to write working code. We ask you to think like the engineer who's read this exact bug before — and to prove it in the prompt.

    01

    No code grading

    We never score syntax. Only diagnosis quality and prompt precision.

    02

    Rewards iteration

    Submit 1 prompt or 5 — refining your ask based on likely output is part of the score.

    03

    Domain-specific ratings

    Your auth-bug rating is tracked separately from your DB or perf rating.
